## 1. 核心概念 / Vision

### 1.1 一句话概述
**[CONFIRMED]** — 多人在线 3D 躲猫猫:藏匿者用环境色伪装、潜行者靠阴影隐身,猎手限时追杀;战场上随机道具与 45s 一次的空投大招,让每一局都是一次欺骗与识别的博弈。

### 1.2 玩家体验核心
| 阵营 | 子职业 | 玩家心理 | 关键感受 |
|------|--------|---------|---------|
| **Prop 队** | 藏匿者 (Chameleon) | "我让世界成为我的皮肤" | 配色博弈、视觉欺骗 |
| **Prop 队** | 潜行者 (Stalker) | "我就是阴影本身" | 时机抉择、暗域博弈 |
| **Hunter** | 猎人 (Hunter) | "哪里不对劲?" | 观察、推理、技能对决 |

### 1.3 设计支柱
1. **伪装即玩法** — 真正的隐藏靠选位 + 行为 + 资源管理,不是速度
2. **每一帧都是推理** — 猎人不停观察怀疑,Prop 不停预判猎人动态
3. **环境即武器** — 光影、颜色、喷涂、阴影是被主动利用的资源
4. **技能有节奏** — 卡牌 + 道具 + 空投,三层资源互相克制,避免单维度最优解

| 维度 | 藏匿者 Chameleon | 潜行者 Stalker | 猎人 Hunter |
|------|------------------|----------------|--------------|

### 2.3 抽签 / 选角流程(v0.3.2 锁定)
- **大厅阶段**:玩家选择 `藏匿者 / 潜行者 / 猎人` 三选一
- **强制比例**:**Hunter : Props = 1 : 3**(v0.3.2 锁定)
- **服务器自动平衡**:玩家选择不符合比例时,服务器调整,优先保证 1:3 比例
- **Props 内部比例**:藏匿者 + 潜行者 = Props 总数,具体分配由玩家选择 / 系统均分
- **典型人数分配表**(v0.3.2 锁定):

| 8 | 2 | 6 | 藏匿者 × 3 + 潜行者 × 3 |
| 12 | 3 | 9 | 藏匿者 × 5 + 潜行者 × 4 |
| 16 | 4 | 12 | 藏匿者 × 6 + 潜行者 × 6 |

## 3. 藏匿者 (Chameleon) — 喷涂系统 + 姿势变形

> **v0.3 重大变更**:
> 1. **颜料无限制** — 取消颜料槽,藏匿者可自由喷涂
> 2. **姿势变形系统** — 玩家可在档案库预设多种身体形态,游戏中通过轮盘切换
> 3. **平衡方式调整** — 用"变形形态库"替代颜料限制作为主要平衡点

### 3.1 核心机制
**通过吸附周围环境色喷涂自身 + 切换预设身体形态,主动实现视觉伪装。**

藏匿者拥有**两套伪装手段**:
- **颜色伪装**:实时喷涂环境色(无消耗,但有"湿润反光"作为反制)
- **形态伪装**:切换到预设的身体形态(瘦长 / 矮胖 / 流线型 / 立方体等)

#### ⚠️ 核心约束(v0.3.1 锁定)

> **藏匿者不可以完美伪装成地图中的可伪装物体。**

具体含义:
- ❌ **不可以直接"变成"地图物体**(不能点击椅子就变成椅子,不能点击纸箱就变成纸箱)
- ❌ **不可以复制地图物体的精确外观**(形状、纹理、颜色完全一致)
- ✅ **只能通过涂装 + 身体比例"模拟"地图物体**:
  - 涂装:吸附物体表面颜色喷到身上(身体本身还是人形)
  - 变形:调整身体比例(身高 / 体重 / 头型 / 四肢),但仍是人形轮廓
- ✅ **"模拟"的程度**:形态匹配度 + 颜色匹配度,综合达到"不易察觉"等级(但仍有破绽)

**设计动机**:
- 经典 Prop Hunt 的"完美变形"过于强大,Prop 一旦选对物体就近乎无敌
- "模拟变形"保留伪装玩法,但给 Hunter 留出识别空间
- 强调"选择位置 + 涂装技术 + 形态匹配"的策略性,而不是"找到物体躲进去"的运气

**游戏影响**:
- Prop 必须主动思考:"这里最适合哪种形态?颜色要怎么涂?站在哪个角度最像?"
- Hunter 知道 Prop 永远是人形,可以根据人形轮廓 + 颜色破绽识别
- 玩法核心从"找物体 → 变进去"变成"选位 + 模拟"

### 3.2 喷涂交互(无颜料限制)
- **触发**:鼠标按住右键 → 持续喷涂(涂装光束对准场景)
- **取色**:光束击中物体表面 → 从该材质 / 贴图采样平均颜色
- **喷涂部位**:躯干 / 四肢 / 头部 / 背面,按光束击中位置就近映射到玩家身体区域
- **颜料量**:**无限制**(v0.3 取消颜料槽)
- **喷涂速度**:1m 半径内 ~0.5s 完成一片区域,完全涂满需要 6-10s 持续对准
- **静止要求**:移动时无法喷涂(喷涂光束需要稳定命中)
- **褪色机制**(平衡手段):
  - **时间褪色**:静置 30s 后颜料轻微褪色(0.5%/s)
  - **天气褪色**:被雨 / 水攻击后 50% 颜料流失
  - **空投雷击**:显形 + 2s 灼烧(颜料加速褪色)

### 3.3 喷涂分段(部位)
| 部位 | 网格 | 喷涂影响 |
|------|------|---------|
| Head | 头部 mesh | 单部位覆盖 |
| Chest | 胸部 + 背部 | 占全身 50% 视觉权重 |
| Limbs | 手臂 + 腿 | 高频识别区,优先涂这里 |
| Bottom | 臀部 | 几乎不会被看到,可不涂 |

### 3.4 喷涂识别度计算(Hunter 视角)
- Hunter 视野看到 Prop 时,后端做颜色差计算:
  - `color_distance(玩家平均色, 环境平均色) < 阈值` → "不易察觉"
  - 否则 → "显眼"
- 颜色差阈值分三档:**Common / Suspicious / Obvious**(影响 UI 高亮)
- **形态加分**:如果形态与环境物体高度匹配,识别度额外 -30%

### 3.5 涂装特效(头脑风暴)
- **光斑效果**:高强度喷涂区域有微弱"湿润反光"(易被识别,警示玩家别喷太过)
- **笔触纹理**:喷涂不是纯色覆盖,而是有笔触方向感,极端靠近能看到
- **气味残留**(Hunter 道具):嗅探道具可以让喷涂过的 Prop 留下"色痕痕迹"
- **饱和度惩罚**:长时间喷涂同一区域,饱和度过度时反而显眼

### 3.6 预先姿势变形系统(NEW in v0.3,REFINED in v0.3.1)

藏匿者除了喷涂颜色,还可以**调整身体比例**(高度 / 体型 / 轮廓),这是 v0.3 加入的核心新系统。**v0.3.1 强化约束**:这套系统仅用于"模拟",不允许"完美变成"地图物体。

#### 3.6.0 v0.3.1 变形边界规则(强制约束)

藏匿者变形有**严格边界**:

| ✅ 允许 | ❌ 不允许 |
|---------|---------|
| 调整身高比例(0.5x - 1.5x) | 复制地图物体的精确外观 |
| 调整身体胖瘦(0.5x - 1.5x) | 替换 mesh 为地图物体 mesh |
| 调整头部大小 / 四肢粗细 | 变成椅子、纸箱、垃圾桶等具体物体 |
| 整体轮廓接近"柱状 / 球状 / 扁平状" | 移除人形基本特征(头、四肢) |
| 保持人形识别度(头 + 躯干 + 四肢) | 改变 mesh 拓扑结构 |

**为什么不允许完美变形**:
1. **避免 Prop 找到物体就无敌** — 经典 Prop Hunt 最大的平衡问题
2. **保留 Hunter 的观察乐趣** — Hunter 知道 Prop 永远是人形,可以识别人形破绽
3. **强调策略性** — 选位 + 涂装 + 形态的组合策略,而不是"一键完美伪装"
4. **降低美术复杂度** — 不需要为每个物体做"Prop 版本 mesh"

**核心提示**:变形是为了"**让人形看起来像在那个位置合理**",不是"**让人形看起来像那个物体**"。

#### 3.6.1 形态预设库(玩家档案)
玩家在游戏外的"角色档案库"中可以预先捏出 **5-10 个身体形态预设**,每个预设包含:
- **身高比例**(0.5x - 1.5x 标准身高)
- **体型**(瘦长 / 标准 / 矮胖 / 流线 / 立方)
- **轮廓特征**(头部比例 / 四肢粗细 / 躯干形状)
- **命名标签**(自定义,如"瘦杆"、"垃圾桶形态")

每个预设"参考"地图上某类物体,但**不是变成那个物体**,只是身体比例接近:
| 形态预设 | **身体参数** | **适合模拟的物体**(参考) |
|---------|------------|----------------------|
| 瘦长型 | 1.5x 高, 0.4x 宽, 小头 | 路灯、旗杆、衣架、走廊尽头的瘦长家具 |
| 矮胖型 | 0.6x 高, 1.4x 宽, 圆头 | 垃圾桶、邮筒、邮箱、矮柜 |
| 立方体型 | 1.0x 高, 1.0x 宽, 方头 | 纸箱附近、货柜旁、机器旁 |
| 流线型 | 0.8x 高, 1.2x 宽, 曲线 | 长椅、沙发、车辆旁 |
| 扁平型 | 0.4x 高, 1.5x 宽, 贴地 | 井盖旁、地砖上、阴影中 |
| 标准人形 | 1.0x 高, 1.0x 宽, 默认头 | 任何位置(用于混合涂装) |

**重要**:这些预设只是"参数",藏匿者**始终保持人形**,只是比例不同。

#### 3.6.2 游戏中变形交互
- **轮盘 UI**:游戏中按 `Q` 打开身体变形轮盘
- **轮盘显示**:8 个形态预设(玩家档案中选 5-10 个里前 8 个)
- **选择**:鼠标点击 / 滚轮选择 / 数字键 1-8 直接切换
- **切换冷却**:**6 秒**(v0.3 锁定,Q5)
- **切换动画**:2s 平滑过渡(挤压 / 拉长),避免瞬变突兀
- **取消选择**:右键 / `Q` 再次按下关闭轮盘
- **变形不消耗颜料**:与喷涂系统独立

#### 3.6.3 变形平衡(强化 v0.3.1)
- **不可变形为完全物体**(v0.3.1 加强):藏匿者只能改身体比例,不能变成具体物体(椅子、纸箱等)
- **必须保持人形基本特征**(v0.3.1 新增):头、躯干、四肢必须可见(不能缩到 0),至少要保留人形轮廓
- **必须用喷涂补充**:形态对了,颜色还要靠喷涂
- **变形有视觉提示**:变形瞬间有粒子效果,Hunter 可能在 5m 内看到
- **不能太频繁**:6s CD 避免无限变形无成本
- **变形仍被识别的破绽**:
  - 人形轮廓(头 + 四肢永远可见)
  - 阴影投射(人形的影子,不是物体的影子)
  - 湿润反光(喷涂部分)
  - 颜色边缘过渡(喷涂笔触)

### 3.7 变形与喷涂的组合策略(REFINED in v0.3.1)

#### 3.7.1 伪装三要素
- **形态匹配**(变形):选择最接近地图物体的身体比例
- **颜色匹配**(喷涂):吸附物体颜色喷到身上
- **位置选择**(选位):站在物体旁边 / 阴影中,利用环境

#### 3.7.2 最佳策略
- 步骤 1:**观察场景**,选择适合的位置(物体旁边 / 阴影中)
- 步骤 2:**变形**到接近物体的形态(如站在邮筒旁 → 矮胖型)
- 步骤 3:**喷涂**对应物体颜色(右键按住,持续 6-10s)
- 步骤 4:**静止不动**,减少破绽(湿润反光、粒子痕迹)

#### 3.7.3 识别度计算(REFINED)
**Prop 永远是"人形 + 涂装 + 变形"组合**,Hunter 通过三个维度识别:

| 维度 | 权重 | 影响因素 |
|------|------|---------|
| **人形破绽** | 40% | 头部可见 / 四肢可识别 / 影子是人形 |
| **颜色匹配** | 35% | 与环境色差(色差越大越显眼) |
| **形态匹配** | 15% | 身体比例与物体的"参考"程度 |
| **行为异常** | 10% | 移动 / 喷涂粒子 / 位置奇怪 |

> 即使形态 + 颜色都完美匹配,**40% 的人形破绽永远无法消除**(除非 Hunter 看不到 / 看不清 Prop)。

#### 3.7.4 极端情况
- **完美模拟不可能**:Hunter 永远能看到 Prop 是"在物体旁边的人形",而不是"物体本身"
- **最佳情况**:Prop 选位极好 + 形态匹配 + 颜色完美 + 完全静止 → Hunter 看到时犹豫 3-5s 才能确认(但只要走到 5m 内必能识破)
- **最差情况**:形态错位 + 颜色明显 + 移动中 → Hunter 50m 外就发现

## 4. 潜行者 (Stalker) — 阴影系统

### 4.1 核心机制
**仅在阴影中隐身,光照即显形。**

### 4.2 阴影判定
- **判定频率**:每 0.2s 一次(或每帧)
- **判定方式**:对玩家中心点向上打 5 条射线(中心 + 前后左右),如果至少 3 条被场景几何遮挡 → 判定为阴影
- **环境光贡献**:DirectionalLight3D / SpotLight3D / 自发光体 / FogGI 都会被检测
- **阴影等级**:
  | 等级 | 条件 | 视觉表现 |
  |------|------|---------|
  | 完全阴影 | 5 条射线全遮挡 | 完全不可见(alpha=0) |
  | 强阴影 | 4 条射线遮挡 | 轮廓模糊(alpha=0.2) |
  | 弱阴影 | 3 条射线遮挡 | 半透明(alpha=0.5) |
  | 明亮 | 2 条以下遮挡 | 正常显形(alpha=1) |

### 4.3 隐身与光照反制
| 光源 | 反制效果 |
|------|---------|
| 探照灯(空投·光) | 范围内直接判定为"明亮",立即显形 |
| 手电筒(道具) | 局部显形 |

## 7. 空投系统(45s 一次,Hunter / Stalker 博弈点)

> **v0.2.1 重大变更**:空投不再是 Hunter 单方面的奖励。**潜行者(Stalker)可以抢先打开空投**,导致 Hunter 失去本次空投。空投现在是双方围绕"地面补给"的战术博弈。

### 7.1 空投基础
- **频率**:每局开始 45s 后第一次,之后每 45s 一次
- **投放方式**:从天空坠落,落点地图随机(但远离 Prop 出生点)
- **状态机**:`天空坠落 → 落地(未激活) → Hunter 拾取 / Stalker 破坏 → 销毁/激活`
- **拾取规则**:
  - ✅ **Hunter** 可拾取,获得空投效果(对 Prop 群生效)
  - ✅ **Stalker(潜行者)** 可抢先打开空投,**销毁空投 + 触发反转效果**(对 Hunter 群不利,见 7.6)
  - ❌ **Chameleon(藏匿者)** 不能接触空投(喷涂职业,接近空投会暴露自身)
  - ⚠️ **Stalker 必须在阴影中**才能悄悄接近空投(被光照见会被 Hunter 拦截)
- **激活**:拾取/破坏后立即生效(无延迟,避免被中断)
- **持续**:30s(部分永久生效)

### 7.1.1 落地与抢先窗口
- **天空坠落**:3s 坠落动画,全场玩家看到信号弹轨迹
- **落地未激活**:落地后进入 10s "双方可接近"窗口
- **争夺状态**:
  - 任何 Hunter 进入 3m 范围 → 自动拾取 → 激活
  - 任何 Stalker(必须阴影中)进入 3m 范围 + 短按 `E` → 抢先破坏 → 销毁 + 反转效果激活
  - Chameleon 进入 3m 范围 → 无效果(只是路过)
- **超时**:10s 内无人拾取 → 空投自动销毁,无效果(避免长期停留)

### 7.2 5 种核心空投

> 每种空投现在有两套效果:**Hunter 激活版** 与 **Stalker 抢先破坏版(反转效果)**。
> 反转设计原则:原来是 Hunter 用来克制 Prop 群,Stalker 抢先后变成"克制 Hunter 群"或"对 Prop 群增益"。

#### 🌧️ **空投 1:雨之洗礼 (Rainfall)**

| 维度 | 内容 |
|------|------|
| **Hunter 激活** | 30s 全场暴雨,所有 Prop 身上的喷涂被冲刷掉 70%,显形为白色本体 |
| **Stalker 抢先** | 空投销毁 → 全场 Hunter 视野被雨水模糊 -30%,持续 20s |
| **反制** | Chameleon 用 `迷彩涂料` 道具抵抗;室内 50% 削弱 |
| **趣味性** | 原本藏匿者最怕,被抢先后 Hunter 反而吃亏 |

| 维度 | 内容 |

### 7.6 空投抢先机制详解(NEW in v0.2.1)

#### 7.6.1 设计动机
原版空投是 Hunter 单方面奖励,容易造成"Hunter 越来越强 → Prop 越来越难赢"的滚雪球效应。
**引入 Stalker 抢先破坏**,让 Prop 阵营有"主动出击"的战术选择,避免单边压制。

#### 7.6.2 抢先流程

```
        天空坠落 (3s)
            │
            ↓
   ┌────────────────────┐
   │  落地未激活(10s)    │ ← 全场可见信号弹轨迹 + 警告音
   │  空投周围 3m        │
   └────────────────────┘
            │
    ┌───────┴────────┐
    │                │
    ↓                ↓
 Hunter 接近    Stalker 接近
 (任意可见)    (必须在阴影中)
    │                │
    ↓                ↓
 ┌──────┐      ┌──────────┐
 │ 拾取 │      │ 短按 E    │
 │ 激活 │      │ 抢先破坏  │
 │ 30s  │      │ 反转 20s  │
 └──────┘      └──────────┘
    │                │
    ↓                ↓
 Hunter 版本    Stalker 反转版
 (克制 Prop)   (克制 Hunter)
```

#### 7.6.3 抢先条件(Stalker 必须满足)
1. **阴影等级 ≥ 3**(完全阴影或强阴影)— 服务器侧实时校验
2. **距空投 ≤ 3m**(自动触发)或 短按 `E`(手动,允许远程观察后再确认)
3. **未达单局抢先次数上限(3 次)**
4. **不是 Chameleon**(藏匿者不能参与空投,职业限制)

#### 7.6.4 抢先失败情况
| 场景 | 结果 |
|------|------|
| Stalker 在光照中接近 3m | 被 Hunter 看到 → Hunter 可攻击 Stalker(暴露位置) |
| Stalker 阴影等级不足 | 接触空投 → 无效,只是路过(不消耗抢先次数) |
| Stalker 抢先次数用尽 | 接触空投 → 无效 |
| Hunter 和 Stalker 同时到 | Hunter 优先(职业优势) |
| 空投 10s 内无人拾取 | 自动销毁,无效果 |

#### 7.6.5 抢先战术价值
- **Stalker 视角**:本来潜行者是"被动挨打"职业,现在有"主动出击"的战术选择。冒险抢空投 → 可能扭转 Hunter 优势
- **Hunter 视角**:不能再无脑去抢空投,得观察空投周围是否有阴影 / 是否安全。增加决策成本
- **Chameleon 视角**:喷涂职业不能抢空投(避免"喷涂+破坏"双任务冲突),专心躲藏
- **整体博弈**:空投落地 10s 内是双方信息博弈高峰(Hunter 看阴影,Stalker 找 Hunter 视野盲区)

#### 7.6.6 反作弊要点
- 阴影判定完全服务器侧(客户端不能伪造)
- 距离校验服务器侧
- 抢先次数记录服务器侧
- 接触事件服务器侧权威(防止双客户端同时触发)
- `[TBD]` 是否允许 Chameleon 用卡牌"涂装炸弹"远程干扰空投(待创意扩展)

#### 7.6.7 抢先成功奖励
- Stalker 抢先成功 → 个人奖励 `+50 金币` + 全队贡献积分 `+10`
- Hunter 拾取成功 → 个人奖励 `+30 金币`
- 抢先失败(被拦截)→ 无奖励,空投仍然存在(Hunter 可拾取)

### 9.1 胜利条件

### 9.2 击杀判定 — AK47 远程射击系统(v0.3 改动)

> **v0.3 重大变更**:Hunter 默认武器从近战改为 **AK47 自动步枪** 远程射击(Q6 锁定)。

#### 9.2.1 AK47 基础参数(v0.3.2 重构)

> **v0.3.2 重大变更**:
> 1. **弹药上限 120 发**(产品锁定)
> 2. **弹药全靠地图拾取**(无初始备弹)
> 3. Hunter 必须先搜刮弹药才能战斗,平衡 Hunter : Props = 1 : 3 的强势 Prop

| 参数 | 数值 |
|------|------|
| 武器类型 | 全自动步枪 |
| 弹匣容量 | 30 发 |
| **总弹药上限** | **120 发**(v0.3.2 锁定) |
| **初始弹药** | **0 发**(必须拾取,v0.3.2 锁定) |

#### 9.2.2 射击机制
- **触发**:`MOUSE_BUTTON_LEFT` 按住 → 自动连发
- **判定方式**:服务器侧 Raycast(从 Hunter 相机中心发出,100m 内)
- **弹道可视化**:**可关闭**(客户端选项,默认开)— 显示弹道光线
- **击中反馈**:被击中方屏幕红色震动 + 受击音效 + 受击者方向指示
- **穿墙**:服务器侧校验,弹道被遮挡则未命中(防作弊)

#### 9.2.3 伤害模型
- **满血 Prop**:100 / 25 = **4 发击杀**
- **爆头**:1.5x 伤害 = 37.5%(3 发爆头击杀)
- **远距离衰减**:60m 后单发 -50% 伤害(2 发难以击杀)
- **击杀确认**:血量归零 → 死亡 → 全场击杀提示
- **击杀有奖励**:Hunter 击杀 +30 金币;Prop 击杀不返还任何奖励

#### 9.2.4 道具 / 卡牌对武器的影响
Hunter 可通过**道具 / 卡牌**替换或增强武器(详见第 5 / 6 节):
- **道具替换**:
  - 消音手枪(单发高伤但低射速)
  - 霰弹枪(近战高伤)
  - 狙击枪(超远距离 + 穿透)
  - 闪光弹发射器
  - 标记飞镖枪
- **卡牌增强**:
  - **强化弹匣** 卡:30s 弹匣容量翻倍
  - **穿甲弹** 卡:5s 内可穿透一个物体
  - **精准瞄准** 卡:5s 内爆头判定区域 +50%
  - **加速换弹** 卡:换弹时间 -50% 持续 20s
  - **弹药搜寻** 卡:20s 内 Hunter 能感知 30m 内弹药包位置

#### 9.2.5 AK47 平衡考虑
- **Prop 反制**(更关键,v0.3.2 Hunter 弹药有限):
  - Hunter 必须先找弹药 → Prop 有更多准备时间
  - Hunter 弹药耗尽时 Prop 可以主动出击
  - 卡牌 `紧急隐蔽` → 受伤时 5s 隐身
  - 卡牌 `静止力场` → 静止 10s 后 8m 范围免疫伤害
- **Hunter 反制 Prop**:
  - 卡牌 `战术扫描` → 5s 内显示 30m 移动 Prop
  - 道具 `红外探测仪` → 30m 热源
  - 道具 `标记飞镖` → 10s 持续位置
  - 道具 `穿甲弹` → 击中可穿透掩体

#### 9.2.6 弹药补给系统(v0.3.2 重构)

> **v0.3.2 核心机制**:Hunter 弹药完全依赖地图拾取,初始为 0 发。

##### 弹药包类型

| 类型 | 容量 | 数量/局 | 分布 |
|------|------|---------|------|
| **小弹药包** | +30 发 | 12-15 个 | 全图散落(常见) |
| **中弹药包** | +60 发 | 6-8 个 | 中等位置(稀有) |
| **大弹药包** | **填满到 120** | 2-3 个 | 关键位置(史诗) |
| **特制弹药包**(可选) | +特殊效果(穿甲/追踪/爆破) | 1-2 个 | 隐藏位置(传奇) |

##### 拾取规则

- **拾取上限**:**总弹药数不能超过 120**(v0.3.2 锁定)
  - 已 90 发拾取 +30 发小包 → 仍是 120(浪费)
  - 已 60 发拾取 +60 发中包 → 120(完美)
  - 已 100 发拾取 +30 发小包 → 仍 120(浪费 10)
- **拾取距离**:走近 1m 自动拾取(同道具系统)
- **视觉提示**:远处能听到 Hunter 拾取音效,Prop 可监听
- **Hunter 共享?**:❌ **不共享**(每位 Hunter 独立弹药槽,鼓励竞争)

##### Hunter 弹药耗尽

- **弹药 = 0 时**:武器变灰色,无法射击
- **回退策略**:
  - 切换为近战(拳头,1 发 10%)
  - 寻找弹药包(可能被 Prop 抢先破坏)
  - 卡牌 `弹药搜寻` → 找弹药包

##### Hunter 死亡(N2 待定)

- 不复活(推荐):Hunter 死了就退出该局
- 复活但弹药减半:死亡时失去一半弹药
- 复活全弹药:死亡不影响弹药

#### 9.2.7 弹药经济平衡(v0.3.2 关键)

> **v0.3.2 平衡核心**:Hunter:Props = 1:3,意味着 Hunter 数量远少于 Props。如果 Hunter 弹药无限,可以轻松击溃 Props。
> 通过 **弹药全靠拾取 + 120 上限**,强制 Hunter:
> 1. 必须花时间搜刮弹药(分散注意力)
> 2. 弹药耗尽时失去战斗力
> 3. Prop 可以监听 Hunter 拾取,预判 Hunter 位置

**典型弹药流(假设 6 Hunter × 18 Props)**:

## 11. 多人网络架构

### 11.1 权威模型
| 操作 | 谁权威 | 校验 |
|------|--------|------|
| 玩家位置 | 服务器 → 全员 | 已有 SceneReplicationConfig,扩频率 |
| 喷涂状态(颜料) | 服务器 | 客户端提交颜料变更 → 服务器广播 |
| 阴影状态(可见度) | **客户端 + 服务器双校验** | 服务器最终决定是否显形 |
| 攻击 | 客户端 → 服务器 raycast | 服务器做命中 |
| 卡牌使用 | 客户端 → 服务器 | 服务器校验 CD + 抽卡池 |
| 道具拾取 / 使用 | 客户端 → 服务器 | 服务器校验距离 + 库存 |
| 空投拾取 / 激活 | 客户端 → 服务器 | 服务器校验职业 + 距离 |
| **空投抢先破坏** | **Stalker 客户端 → 服务器** | **服务器校验:职业=Stalker + 阴影等级 ≥ 3 + 距离 ≤ 3m + 抢先次数 < 3** |
| 聊天 | 客户端 → 服务器 → 全员 | 已有 |
| 胜负 | 服务器单方面 | — |

### 11.2 同步优化
- **AOI**:客户端只关心附近 50m Props(其它降频)
- **LOD**:远距离 Props 用位置 + 朝向,不需要喷涂细节
- **批量事件**:空投生效、击杀提示等单次事件,不走 SceneReplication
- **关键帧 / 插值**:复用 Godot 4 内置机制

### 11.3 反作弊
- **喷涂伪装**:客户端不能说自己喷了什么色,服务器侧基于位置 + 时间戳校验
- **阴影隐身**:服务器侧每帧算光照,客户端显示
- **攻击射线**:完全服务器侧
- **道具拾取**:距离 + 库存双重校验
- **卡牌抽卡**:服务器侧 RNG,客户端只接收结果
- **空投抢先**:服务器侧权威校验四要素(职业 / 阴影 / 距离 / 次数),客户端无法伪造

### 11.4 卡牌系统网络流(扩展 Hunter)

Hunter 卡牌流程与 Prop 卡牌对称:
```
Hunter 出生 → 服务器从 Hunter 卡池抽 2 次各 3 张 → 选 2 张保留
使用:Client → Server.rpc("use_hunter_card", card_id) → Server 校验 → 触发
```

## 11.5 视角切换系统(NEW in v0.3)

> **v0.3 新增需求**:藏匿者"藏好之后"可以选择切换**自由视角**和**玩家视角**,查看其他玩家的视角内容(包括 Hunter、潜行者、其他藏匿者的视角)。

### 11.5.1 触发条件

**"藏好"定义**(满足以下任一):
- 喷涂度达到 80% 以上
- 进入阴影等级 ≥ 3(对潜行者)
- 准备阶段剩余 < 10s(强制可切换)
- 主动按 `Tab` 键强制进入观察模式

### 11.5.2 视角模式

#### 模式 A:自由视角 (Free Cam)
- **操作**:WASD 移动 + 鼠标控制视角 + `Shift` 加速 + `Space` 上升 + `Ctrl` 下降
- **可用性**:**无限制**,任何玩家可切换
- **可见性**:**仅自己可见**,其他玩家看不到自由视角玩家的位置
- **限制**:不能离战场太远(半径 200m 内)

#### 模式 B:玩家视角 (Spectator)
- **操作**:Tab 打开玩家列表 → 鼠标选择目标玩家 → 视角锁定到该玩家
- **可选目标**:
  - **Hunter 玩家**(可看 Hunter 看到的视角,看到 AK47 弹道)
  - **潜行者玩家**(可看潜行者看到的视角,包含阴影判定)
  - **其他藏匿者玩家**(可看其他藏匿者看到的视角,看他们喷涂后的样子)
- **可见性**:**仅自己可见**,被观察玩家不知道被观察
- **限制**:最多 5 个视角候选(轮盘 / 列表展示)

### 11.5.3 切换交互

```
按 Tab 键 → 视角切换菜单
  ├── [当前视角]: 第一人称 (Chameleon 自己)
  ├── [自由视角]: 自由相机
  └── [玩家视角] (子菜单):
        ├── Hunter #1 (Username)
        ├── Hunter #2 (Username)
        ├── Stalker #1 (Username)
        ├── Chameleon #1 (Username)
        └── Chameleon #2 (Username)
```

- **键位**:`Tab` 打开 / 关闭切换菜单
- **键位**:`F1-F5` 快速切换到预设的 5 个观察目标
- **键位**:`Esc` 退出观察,回到第一人称

### 11.5.4 视角下的能力限制

| 视角 | 可以做的 | 不可以做的 |
|------|---------|----------|
| 第一人称(自己) | 全部 | — |
| 自由视角 | 看任何位置 | 攻击 / 喷涂 / 拾道具 |

### 11.5.5 网络同步要点

- **视角数据**:**纯本地**(其他玩家不知道你在看谁的视角)
- **服务器侧**:**无需同步**,纯客户端行为
- **反作弊**:被观察玩家不能"识别"是否被观察
- **死亡处理**:被观察玩家死亡后,观察者自动跳到下一个玩家 / 自由视角

### 11.5.6 UI 设计

#### 观察模式 HUD 元素
- **顶部横幅**:当前观察对象 + 切换提示

- **藏好才能切换**:避免 Prop 战时就切走,失去游戏性

```gdscript
# shape_shift_system.gd - 变形核心
class_name ShapeShiftSystem
extends Node

const SHIFT_COOLDOWN: float = 6.0  # Q5 锁定
var current_preset_index: int = -1
var is_shifting: bool = false
var cooldown_remaining: float = 0.0

signal shift_started(preset_index: int)
signal shift_completed(preset_index: int)
signal shift_failed(reason: String)

func try_shift(preset_index: int, player: Character) -> bool:
    if cooldown_remaining > 0.0:
        shift_failed.emit("cooldown")
        return false
    if is_shifting:
        shift_failed.emit("already_shifting")
        return false

    current_preset_index = preset_index
    is_shifting = true
    shift_started.emit(preset_index)

    # 2s 平滑过渡动画
    var tween = player.create_tween()
    var preset = ProfileManager.get_preset(preset_index)
    tween.tween_property(player, "scale", Vector3(
        preset.width_scale, preset.height_scale, preset.width_scale
    ), 2.0)
    await tween.finished

    is_shifting = false
    cooldown_remaining = SHIFT_COOLDOWN
    shift_completed.emit(preset_index)
    return true

func _process(delta):
    if cooldown_remaining > 0.0:
        cooldown_remaining -= delta
```

```gdscript
# weapon_system.gd - AK47 武器系统
class_name WeaponSystem
extends Node3D

const MAGAZINE_SIZE: int = 30
const RESERVE_AMMO: int = 90
const FIRE_RATE: float = 0.1  # 600 RPM
const DAMAGE_PER_BULLET: float = 25.0
const HEADSHOT_MULTIPLIER: float = 1.5
const RELOAD_TIME: float = 2.5
const MAX_RANGE: float = 60.0

var current_ammo: int = MAGAZINE_SIZE
var reserve_ammo: int = RESERVE_AMMO
var is_firing: bool = false
var is_reloading: bool = false

signal ammo_changed(current: int, reserve: int)
signal fired(hit_position: Vector3, hit_target: Node)
signal reload_started
signal reload_completed

@rpc("any_peer", "call_local", "reliable")
func request_fire(aim_direction: Vector3, shooter_position: Vector3):
    if not multiplayer.is_server(): return
    if current_ammo <= 0 or is_reloading: return

    current_ammo -= 1
    ammo_changed.emit(current_ammo, reserve_ammo)

    # 服务器侧 raycast 命中判定
    var space_state = get_world_3d().direct_space_state
    var query = PhysicsRayQueryParameters3D.create(
        shooter_position,
        shooter_position + aim_direction * MAX_RANGE
    )
    var result = space_state.intersect_ray(query)
    if result and result.collider.has_method("take_damage"):
        var is_headshot = result.collider.is_in_group("head")
        var damage = DAMAGE_PER_BULLET * (HEADSHOT_MULTIPLIER if is_headshot else 1.0)
        result.collider.take_damage.rpc_id(1, damage, is_headshot)
        fired.emit(result.position, result.collider)
```

```gdscript
    mode_changed.emit(current_mode, -1)
    Input.mouse_mode = Input.MOUSE_MODE_CAPTURED

func enter_player_view(target_player_id: int):
    spectated_player_id = target_player_id
    current_mode = Mode.PLAYER_VIEW
    mode_changed.emit(current_mode, target_player_id)

func exit_to_first_person():
    current_mode = Mode.FIRST_PERSON
    spectated_player_id = -1
    mode_changed.emit(current_mode, -1)

func is_spectating() -> bool:
    return current_mode != Mode.FIRST_PERSON

# 注:此系统为纯客户端行为,服务器不参与
```

### 15.0.4 Hunter 准备室机制(v0.3.3 锁定)

> **N6 锁定答案**:准备阶段(120s)Hunter **只能在准备室中活动**,不可进入主战场地图。

#### 准备室设计
- **位置**:地图边界外的独立空间(可通过传送门/隧道视觉连接,但物理隔离)
- **大小**:容纳所有 Hunter 的最小空间(8 人 ≈ 30×30m,24 人 ≈ 60×60m)
- **设施**:
  - Hunter 出生点(每位 Hunter 一个固定位置)
  - 准备弹药箱(每位 Hunter 出生时 0 发,但有 1 个"参考弹药箱"显示 AK47 模型)
  - 战场观察窗(可观察主战场,但不能攻击)
  - Hunter 之间可以聊天交流
- **限制**:
  - Hunter 不能跨越准备室边界进入主地图
  - 准备室门在 120s 准备结束时打开
  - Hunter 进入主地图后无法返回准备室

#### Prop 准备阶段
- **位置**:主战场地图内的 Prop 出生区
- **设施**:
  - 自由跑动选择藏匿点
  - 喷涂 + 变形(受 6s CD 限制)
  - Hunter 完全看不到(准备室隔离)
- **限制**:
  - 不能跨越到 Hunter 准备室
  - 不能跨越地图边界

#### 阶段转换
```
T = 0s    所有玩家进入准备
T = 0-120s Hunter 在准备室;Props 在战场出生区
T = 120s  准备结束,Hunter 门打开,冲向战场;Props 进入躲藏状态
T > 120s  PLAY 阶段开始(单局时长 10min 默认)
```
